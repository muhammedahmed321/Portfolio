Project Overview

The E-Book Management System is a dynamic web application designed to simplify the process of managing and accessing digital books. It allows the admin to easily add, update, and delete books, while users can browse and view available eBooks through a clean and responsive interface. The project integrates a fully functional backend that manages data efficiently and ensures secure authentication for users and administrators. This system aims to provide an organized platform for book management and seamless communication through a built-in Contact Form. Challenges & Solutions

üß© Challenge 1: Database Connection & Error Handling

Issue: Initially faced difficulty in connecting the website to the MySQL database and managing error responses. Solution: Used mysqli_connect() with proper exception handling and tested queries step-by-step to ensure smooth data operations.
üß© Challenge 2: Securing Login System
Issue: Login and logout sessions were not maintaining properly, leading to unauthorized access. Solution: Implemented PHP sessions and conditional redirects to protect admin pages from direct URL access.
üß© Challenge 3: CRUD Operations Functionality
Issue: Updating and deleting records caused inconsistencies due to missing IDs in the query. Solution: Implemented hidden form fields and URL parameters to correctly identify book IDs for update/delete actions.
üß© Challenge 4: Template Customization
Issue: Integrating the backend code with a pre-built HTML/CSS template caused layout and routing issues. Solution: Structured files properly and embedded PHP blocks within the HTML for seamless backend integration.
üß© Challenge 5: Hosting and Environment Setup
Issue: Migrating the project between different environments (localhost and live server) caused configuration errors. Solution: Used relative paths, updated database credentials dynamically, and tested the project using XAMPP before deployment.
